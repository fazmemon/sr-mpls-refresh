Current state: PE1 configured; PE1↔P1 link wasn't passing traffic (see link-binding fix below).

FIX #1 — link binding (do this whenever a directly-connected ping fails despite Up/Up): EVE only binds a link to a node's NIC at boot. If you started nodes then drew links (or redrew them), Stop all nodes → confirm links present → Start all so every link binds before boot. Interfaces showing Up/Up but unable to ping the neighbour = this.
FIX #2 — the /30 addressing bug (cause of the ASBR1 commit failure): every point-to-point link is a /30 = exactly 2 usable hosts, .1 and .2 (.0 = network, .3 = broadcast). The earlier doc used "last-loopback-octet" (e.g. .3, .8) for link IPs — those are NOT valid /30 hosts → "Invalid host address or mask". Every link below now uses .1/.2. Loopbacks stay 10.0.0.N/32 (those are fine — /32).
